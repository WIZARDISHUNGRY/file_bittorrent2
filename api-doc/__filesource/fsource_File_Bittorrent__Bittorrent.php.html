    29. 

  29.     *
    30. 

  30.     * Info on the .torrent file format
    31. 

  31.     *
    32. 

  32.     * [http://www.monduna.com/bt/faq.html]
    33. 

  33.     * BEncoding is a simple, easy to implement method of associating
    34. 

  34.     * data types with information in a file. The values in a torrent
    35. 

  35.     * file are bEncoded.
    36. 

  36.     * There are 4 different data types that can be bEncoded:
    37. 

  37.     * Integers are prefixed with an i and terminated by an e. For
    38. 

  38.     * example, 123 would bEcode to i123e, -3272002 would bEncode to
    39. 

  39.     * i-3272002e.
    40. 

  40.     * Strings are prefixed with their length followed by a colon.
    41. 

  41.     * For example, "Monduna" would bEncode to 7:Monduna and "BitTorrents"
    42. 

  42.     * would bEncode to 11:BitTorrents.
    43. 

  43.     * Lists are prefixed with a l and terminated by an e. The list
    44. 

  44.     * should contain a series of bEncoded elements. For example, the
    45. 

  45.     * list of strings ["Monduna", "Bit", "Torrents"] would bEncode to
    46. 

  46.     * l7:Monduna3:Bit8:Torrentse. The list [1, "Monduna", 3, ["Sub", "List"]]
    47. 

  47.     * would bEncode to li1e7:Mondunai3el3:Sub4:Listee
    48. 

  48.     * Dictionaries are prefixed with a d and terminated by an e. They
    49. 

  49.     * are similar to list, except that items are in key value pairs. The
    50. 

  50.     * dictionary {"key":"value", "Monduna":"com", "bit":"Torrents", "number":7}
    51. 

  51.     * would bEncode to d3:key5:value7:Monduna3:com3:bit:8:Torrents6:numberi7ee
    52. 

  52.     *
    53. 

  53.     * [http://animesuki.com/dev.php]
    54. 

  54.     * BEncode is the file format for ".torrent" files. A few interesting things can
    55. 

  55.     * be retrieved from ".torrent" files, namely the tracker URL, the filename and
    56. 

  56.     * filesize of the file it's pointing to and checksums for all the individual pieces
    57. 

  57.     * of the file.
    58. 

  58.     * If you calculate the SHA-1 hash of the "info" part of a ".torrent" file you can
    59. 

  59.     * also get the "infohash" value shown on the information page of a tracker. This is
    60. 

  60.     * useful if you want to get the number of sources and clients for a ".torrent" file.
    61. 

  61.     * Please note that the sha1() function used for calculating the infohash requires
    62. 

  62.     * PHP 4.3.0 or newer.
