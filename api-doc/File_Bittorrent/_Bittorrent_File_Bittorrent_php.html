Info on the .torrent file format

[http://www.monduna.com/bt/faq.html] BEncoding is a simple, easy to implement method of associating data types with information in a file. The values in a torrent file are bEncoded. There are 4 different data types that can be bEncoded: Integers are prefixed with an i and terminated by an e. For example, 123 would bEcode to i123e, -3272002 would bEncode to i-3272002e. Strings are prefixed with their length followed by a colon. For example, "Monduna" would bEncode to 7:Monduna and "BitTorrents" would bEncode to 11:BitTorrents. Lists are prefixed with a l and terminated by an e. The list should contain a series of bEncoded elements. For example, the list of strings ["Monduna", "Bit", "Torrents"] would bEncode to l7:Monduna3:Bit8:Torrentse. The list [1, "Monduna", 3, ["Sub", "List"]] would bEncode to li1e7:Mondunai3el3:Sub4:Listee Dictionaries are prefixed with a d and terminated by an e. They are similar to list, except that items are in key value pairs. The dictionary {"key":"value", "Monduna":"com", "bit":"Torrents", "number":7} would bEncode to d3:key5:value7:Monduna3:com3:bit:8:Torrents6:numberi7ee

[http://animesuki.com/dev.php] BEncode is the file format for ".torrent" files. A few interesting things can be retrieved from ".torrent" files, namely the tracker URL, the filename and filesize of the file it's pointing to and checksums for all the individual pieces of the file. If you calculate the SHA-1 hash of the "info" part of a ".torrent" file you can also get the "infohash" value shown on the information page of a tracker. This is useful if you want to get the number of sources and clients for a ".torrent" file. Please note that the sha1() function used for calculating the infohash requires PHP 4.3.0 or newer.
